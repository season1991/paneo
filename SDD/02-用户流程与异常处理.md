# Paneo — 用户流程与异常处理

> 配套 [01-产品定义与范围.md](01-产品定义与范围.md)，记录"从打开到完成一次查询"的完整路径。
> 每个步骤标注：正常流程 / 异常情况 / 处理方式。

---

## 阶段一：启动

### 正常流程
1. 用户双击 Paneo 图标
2. 程序启动，显示主窗口
3. 读取本地配置文件（已保存的网站列表 + 布局设置）
4. 按配置创建分屏布局，逐个加载 webview
5. 每个 webview 开始加载对应的 AI 网站 URL
6. 全局输入框就绪，等待用户输入

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 配置文件不存在 | 首次启动，无任何配置 | 使用默认配置（预置 AI 列表，2列布局） |
| 配置文件损坏 | JSON 解析失败 | 弹窗提示"配置已重置"，使用默认配置覆盖 |
| 无网络连接 | webview 加载失败 | 面板内显示"无法连接，请检查网络"+ 重试按钮 |
| AI 网站本身宕机 | 页面加载超时（>30s） | 面板内显示"网站暂时不可用"+ 重试按钮 |
| 网站 SSL 证书错误 | Electron 默认拦截 | 面板内显示"连接不安全"，不提供"继续访问"（安全考虑） |

## 阶段二：登录 AI 网站

### 正常流程
1. 用户在某个面板中看到 AI 网站的登录页面
2. 用户在该面板内自行完成登录（输入账号密码、扫码等）
3. 登录成功后，该面板进入 AI 对话界面
4. 登录状态保存在该面板的独立 session 中
5. 其他面板不受影响，各自独立登录

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 登录后页面跳转到非对话页 | 如跳到主页、设置页等 | 不干预，用户自行导航到对话页 |
| 需要验证码/二次验证 | 弹出验证码或短信验证 | 不干预，用户在面板内自行完成 |
| 登录态过期 | 下次启动时需要重新登录 | 正常现象，不处理。用户重新登录即可 |
| 网站检测到 webview 并拒绝 | 显示"请使用浏览器打开"等 | 面板内显示原始提示，不伪装 User-Agent（见 N6） |
| 网站要求安装 App | 弹出"下载 App"遮罩 | 不干预，用户自行关闭遮罩或放弃该网站 |

---

## 阶段三：输入问题

### 正常流程
1. 用户在全局输入框中输入问题文字
2. 按下回车键，或点击发送按钮
3. 程序遍历所有已加载的面板
4. 对每个面板执行：
   a. 找到该网站的输入框元素（按选择器优先级列表匹配）
   b. 将问题文字填入输入框
   c. 触发 input 事件（确保 React/Vue 等框架感知到值变化）
   d. 模拟按下 Enter 键，或点击发送按钮
5. 各 AI 网站开始生成回答
6. 全局输入框清空，等待下一次输入

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 全局输入框为空时按回车 | 无内容 | 忽略，不发送 |
| 某个面板未加载完成 | 网站还在加载中 | 跳过该面板，不注入。全局输入框旁显示"跳过 1 个未就绪的网站" |
| 某个面板未登录 | 显示的是登录页 | 跳过该面板。同上提示 |
| 找不到输入框元素 | 选择器全部未匹配 | 跳过该面板。该面板右上角显示小红点，hover 提示"未能找到输入框" |
| 注入文字后未触发发送 | 网站前端框架特殊处理 | 重试一次（先尝试 Enter，失败则尝试点击发送按钮）。仍失败则放弃，显示提示 |
| 网站弹出确认对话框 | 如"确定发送？" | 不干预，用户自行在面板内确认 |
| 同时注入的网站过多导致卡顿 | 界面响应变慢 | 串行注入（一个接一个），而非并行。间隔 200ms |

---

## 阶段四：查看回答

### 正常流程
1. 用户逐个面板滚动查看各 AI 的回答
2. 用户可在任意面板内继续对话（点击、滚动、追问）
3. 各面板操作完全独立

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| AI 回答需要很长时间 | 某个 AI 还在生成 | 不干预，用户自行等待 |
| AI 网站报错 | 如"服务器错误" | 不干预，用户可点击刷新按钮重试 |
| 面板内容太多导致内存占用高 | 6 个 webview 同时运行 | 可接受。后期可优化（冻结不可见面板），第一版不做 |

---

## 阶段五：调整布局

### 正常流程
1. 用户点击布局切换按钮（2列/3列）
2. 分屏布局重新排列
3. 各 webview 保持当前状态，不重新加载

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 切换布局时 webview 闪烁 | 重排导致重绘 | 可接受，不处理 |
| 减少列数后面板变小 | 网站可能响应式变化 | 不干预，用户自行在面板内缩放 |

---

## 阶段六：管理网站

### 正常流程

**添加网站：**
1. 用户点击"添加网站"按钮
2. 弹出对话框，输入：网站名称、URL
3. 可选：高级模式填写输入框选择器（不填则自动检测）
4. 点击确认，新面板出现在分屏中
5. 配置保存到本地 JSON

**删除网站：**
1. 用户点击面板右上角的"×"按钮
2. 弹出确认"确定移除 xxx？"
3. 确认后关闭该 webview，从分屏中移除
4. 配置保存到本地 JSON

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 添加的 URL 格式错误 | 如缺少协议头 | 自动补全 `https://` |
| 添加的 URL 无法访问 | webview 加载失败 | 仍然添加成功，面板显示加载错误（用户可能是暂时断网） |
| 添加重复的 URL | 与已有网站相同 | 允许重复（用户可能想开两个不同账号的同网站 — 虽然第一版不做，但不禁止） |
| 删除正在使用的网站 | 用户正在对话中 | 确认后直接关闭，不挽留 |

## 选择器失效的特殊说明

这是**最常见、最可能发生的异常**。AI 网站随时可能更新前端代码，导致预置的 CSS 选择器失效。

**处理策略：**
- 程序不自动更新选择器
- 当注入失败时，面板显示提示："自动输入失败，请在该网站内手动输入"
- 用户可以通过"编辑网站"功能手动更新选择器
- 预置选择器的更新随软件版本发布

**这是第一版的已知限制，不修。**

## 阶段七：网站编辑器 — 自动分析

当用户在网站编辑器中点击 [分析] 按钮时，触发对当前网站页面结构的自动检测，以发现注入所需的 CSS 选择器。

### 正常流程

1. 用户在网站编辑器中确认目标 URL
2. 点击 [分析] 按钮
3. 分析引擎在 webview 中加载目标页面（如已加载则直接使用当前页面）
4. 引擎遍历 DOM，扫描所有 `<textarea>`、`<input type="text">`、`contenteditable` 元素
5. 对每个候选元素评分排序（按元素类型、可见性、位置、属性特征等综合判断），选出最佳匹配
6. 对候选发送按钮执行类似扫描：查找 `button` / `[role="button"]` 元素，按"发送"/"submit" 语义筛选
7. 综合排序，选出最佳输入框和发送按钮选择器
8. 前端展示分析结果，用户点击 [使用此结果] 将选择器写入 SiteConfig
9. 编辑器对已选选择器执行一次注入验证（`executeJavaScript` 模拟填入并检查），显示"可注入"/"不可注入"状态

### 异常情况

| 异常 | 表现 | 处理 |
|------|------|------|
| 页面加载超时 | 分析中状态持续超过 15 秒 | 显示"页面加载超时"，提供 [重试] 和 [手动配置] 按钮 |
| 页面禁止 iframe/webview 嵌入 | iframe 显示空白或拒绝加载 | 显示 fallback 提示，提示用户在独立窗口中登录后重试，或直接使用手动配置 |
| 未找到任何输入框候选 | 扫描结果为空 | 显示"未检测到输入框"，建议用户手动配置选择器 |
| 检测到多个候选但都不确定 | 评分最高的候选元素不明确 | 直接展示最佳匹配结果，提示用户如有问题可手动配置 |
| 发送按钮未找到 | 无 button 候选 | 输入框选择器仍然展示，发送按钮行显示"未检测到"，注入将回退到模拟 Enter 键 |
| 注入验证失败 | 选定选择器执行注入后文本未出现 | 显示"不可注入"警告，建议用户调整选择器或使用手动配置 |
| 网站需要登录才能进入对话页 | 分析页面是登录页 | 检测到常见登录表单特征后显示"请先登录，登录后重新分析" |

### 自动分析 vs 手动配置的决策逻辑

```text
用户打开网站编辑器
      │
      ▼
该网站已有 selectors？ ──是──▶ 显示当前选择器 + [重新分析] 按钮
      │
     否
      ▼
是否为预置网站？ ──是──▶ 使用预置选择器，显示 [分析] 按钮供验证
      │
     否
      ▼
显示 [分析] 按钮，引导用户执行自动分析
      │
      ▼
分析成功？ ──是──▶ 显示结果 + [使用此结果] + [手动配置]
      │
     否
      ▼
显示错误 + [重试] + [手动配置]
```
