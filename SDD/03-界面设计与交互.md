## 1. 设计系统（Design Tokens）

### 1.1 色彩

| Token | 值 | 用途 |
|-------|-----|------|
| `--bg` | `oklch(97% 0.004 250)` | 页面底色 |
| `--surface` | `oklch(100% 0 0)` | 卡片/面板/导航栏背景 |
| `--fg` | `oklch(20% 0.018 250)` | 主文字 |
| `--muted` | `oklch(50% 0.016 250)` | 次要文字/占位符 |
| `--border` | `oklch(89% 0.006 250)` | 边框/分割线 |
| `--accent` | `oklch(58% 0.16 145)` | 主强调色（绿色） |
| `--accent-dim` | `oklch(58% 0.16 145 / 0.12)` | 强调色低透明度 |
| `--danger` | `oklch(55% 0.22 25)` | 错误/删除 |
| `--warn` | `oklch(68% 0.16 85)` | 警告 |
| `--shell` | `oklch(16% 0.012 250)` | 深色壳体（预留） |

**AI 品牌色**

| AI | 变量 | 值 | 语义 |
|----|------|-----|------|
| ChatGPT | `--ai-chatgpt` | `oklch(55% 0.16 145)` | 绿色 |
| Claude | `--ai-claude` | `oklch(55% 0.14 60)` | 琥珀色 |
| Gemini | `--ai-gemini` | `oklch(56% 0.18 250)` | 蓝色 |
| DeepSeek | `--ai-deepseek` | `oklch(55% 0.16 250)` | 蓝色 |

每个面板通过 `data-ai` 属性关联 `--panel-accent` 变量，用于顶部色条、消息气泡头像等。

### 1.2 字体

| Token | 值 | 用途 |
|-------|-----|------|
| `--font-body` | `-apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI', system-ui, sans-serif` | 正文/UI 文字 |
| `--font-mono` | `'JetBrains Mono', 'IBM Plex Mono', ui-monospace, Menlo, monospace` | 状态标签/URL/代码 |

基准字号：`13px`，行高 `1.5`。

### 1.3 间距与圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `--r` | `8px` | 标准圆角（按钮/输入框） |
| `--r-lg` | `12px` | 大圆角（面板卡片） |
| `--gap` | `10px` | 面板间距 |
| `--header-h` | `52px` | 顶部导航栏高度 |
| `--input-h` | `68px` | 底部输入栏高度 |

## 2. 整体布局（App Shell）

应用窗口采用 CSS Grid 三行布局，垂直方向分为三个区域：

```
┌─────────────────────────────────────────────────────────────┐
│                        顶部导航栏 (52px)                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                      面板网格区域 (1fr)                       │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                      统一输入栏 (68px)                        │
└─────────────────────────────────────────────────────────────┘
```

```
grid-template-rows: 52px 1fr 68px;
height: 100vh;
```

- 整体不可滚动，三个区域填满视窗
- 背景色：`--bg`
- 面板网格区域内边距：`10px`
- 区域之间无额外间隙

## 3. 顶部导航栏

导航栏采用 Flex 水平布局，白色背景，底部 1px 边框 + 微阴影。

```
┌─────────────────────────────────────────────────────────────┐
│  ▦ AI 多开浏览器  │  [2 列] [3 列]     ● 4 个 AI 已连接  [＋][⚙] │
└─────────────────────────────────────────────────────────────┘
```

### 区域构成

| 区域 | 内容 | 说明 |
|------|------|------|
| 品牌标识 | 四格图标 SVG + "AI 多开浏览器" | 14px 粗体，不换行 |
| 分隔线 | 1px × 24px 竖线 | `--border` 颜色 |
| 布局切换 | 两个带图标的按钮：`2 列` / `3 列` | 当前激活态用 `--accent-dim` 背景 + `--accent` 文字和边框 |
| 弹性间隔 | `flex: 1` | 将右侧内容推到最右 |
| 状态指示 | 绿色圆点 (6px) + "N 个 AI 已连接" | 11px 等宽字体，`--muted` 颜色 |
| 操作按钮 | `＋` 添加面板 / `⚙` 设置 | 32×32 图标按钮，hover 时 `--accent-dim` 背景 |

### 布局切换按钮

- 尺寸：高度 32px，水平内边距 12px
- 圆角：`--r` (8px)
- 字体：12px 等宽字体
- 默认态：白底，`--border` 边框，`--muted` 文字
- Hover：边框变为 `--accent`，文字变为 `--fg`
- 激活态：`--accent-dim` 背景，`--accent` 边框和文字，600 字重

### 操作按钮（图标按钮）

- 尺寸：32 × 32px
- 无边框，透明背景
- 图标：16px，`--muted` 颜色
- Hover：`--accent-dim` 背景，`--fg` 图标色
- `[＋]` 弹出"添加网站"对话框
- `[⚙]` 弹出设置面板

## 4. 面板网格

面板区域使用 CSS Grid 自动排列，位于导航栏下方、输入栏上方，填充剩余空间。

### 网格规则

| 布局 | `grid-template-columns` | 面板排列 |
|------|------------------------|---------|
| 2 列 | `1fr 1fr` | 左→右，满一行换行 |
| 3 列 | `1fr 1fr 1fr` | 左→右，满一行换行 |

- 面板间距：`--gap` (10px)
- 区域内边距：`--gap` (10px)
- 面板不足时，空位留白，不拉伸
- 切换布局时过渡动画：`0.3s cubic-bezier(0.4, 0, 0.2, 1)`

## 5. 面板详解

单个面板模拟浏览器 mini chrome，顶部是 Paneo 绘制的控制栏，下方为 webview 内容区。

### 5.1 面板结构

```
┌─ 2px 品牌色条 ─────────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com            │  [↻] [⤢]     │  ← 面板控制栏 (38px)
├───────────────────────────────────────────────────────────┤
│                                                           │
│                                                           │
│                    webview 内容区域                        │
│                  (AI 网站原生界面)                          │
│                                                           │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

### 5.2 面板控制栏

高度 38px，背景 `oklch(97% 0.003 250)`，底部 1px `--border` 边框。

| 元素 | 样式 | 功能 |
|------|------|------|
| 品牌色条 | 面板顶部 2px 实线，颜色为 `--panel-accent` | 视觉标识当前 AI |
| 交通灯圆点 | 3 个 9px 圆，分别为红/黄/绿色 | 纯装饰，模拟浏览器窗口控件 |
| URL 栏 | `flex: 1`，高度 24px，左侧地球图标 (10px, `--accent` 色)，显示域名 | 展示当前网站地址，不可编辑 |
| `[↻]` 刷新按钮 | 24×24px，刷新图标 (12px) | 刷新当前页面 |
| `[⤢]` 展开按钮 | 24×24px，对角箭头图标 (12px) | 将该面板全屏展开（焦点模式） |

按钮默认 `--muted` 色，hover 时背景 `oklch(0% 0 0 / 0.06)`，文字 `--fg`。

### 5.3 面板交互态

| 场景 | 表现 |
|------|------|
| 默认 | 1px `--border` 边框 |
| Hover | 阴影 `0 4px 16px oklch(0% 0 0 / 0.08)`，边框色变为 `--panel-accent` 30% 透明度 |
| 发送闪烁 | 内边框 2px `--accent` 色闪现 0.8s 后消失 |

## 6. 面板状态

每个面板有 5 种状态，同一时间只显示一种。状态由 Paneo 自动检测并切换。

### 状态 A：加载中

面板内容区显示居中加载提示：

```
┌─────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com  [↻][⤢] │
├─────────────────────────────────────────┤
│                                         │
│                                         │
│              ◌ 加载中...                 │
│                                         │
│                                         │
└─────────────────────────────────────────┘
```

- 居中排列，`--muted` 颜色
- 图标透明度 0.4
- 文字 12px，标签 11px 等宽字体

### 状态 B：正常运行

```
┌─────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com  [↻][⤢] │
├─────────────────────────────────────────┤
│                                         │
│         (网站正常显示)                    │
│                                         │
└─────────────────────────────────────────┘
```

### 状态 C：加载失败

```
┌─────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com  [↻][⤢] │
├─────────────────────────────────────────┤
│                                         │
│             ✕ 加载失败                   │
│                                         │
│       可能原因：网络中断 / 网站不可用      │
│                                         │
│            [  重新加载  ]                │
│                                         │
└─────────────────────────────────────────┘
```

### 状态 D：注入失败（选择器未匹配）

网站正常显示，但自动输入功能失效。面板控制栏右上角出现红色小圆点 `●`（hover 提示"自动输入不可用"）。

内容区上方叠加横幅提示：

```
┌─────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com  [↻][⤢] ●│  ← 红色指示点
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │ ⚠ 未能自动输入，请手动在此输入框中   │ │
│ │   粘贴你的问题。             [关闭]  │ │
│ └─────────────────────────────────────┘ │
│                                         │
│         (网站正常显示)                    │
│                                         │
└─────────────────────────────────────────┘
```

- 红点 hover 显示 tooltip："自动输入不可用"
- 横幅出现 5 秒后自动收起，或点击关闭
- 用户可正常手动操作该面板

### 状态 E：未就绪（未登录 / 页面非对话页）

面板控制栏右上角出现空心圆 `○`（表示"此面板暂不可接收输入"）。

```
┌─────────────────────────────────────────┐
│  ● ● ●  │  🌐 chat.openai.com  [↻][⤢] ○│  ← 空心指示圆
├─────────────────────────────────────────┤
│                                         │
│     (网站显示的是登录页或其他页面)         │
│                                         │
└─────────────────────────────────────────┘
```

- 用户完成登录或导航到对话页后，自动变为状态 B
- 空心圆消失

## 7. 统一输入栏

位于窗口底部，白色背景，顶部 1px 边框。水平 Flex 布局，垂直居中。

```
┌─────────────────────────────────────────────────────────────┐
│  统一输入  │ 🔍 输入问题，同时发送到所有 AI...    Enter ↵ │ 🗑 │  发送  │
└─────────────────────────────────────────────────────────────┘
```

### 区域构成

| 区域 | 样式 | 说明 |
|------|------|------|
| 标签 | 10px 等宽字体，`--muted` 色，大写，`letter-spacing: 0.08em` | "统一输入"文字标识 |
| 输入区 | `flex: 1`，高度 40px，`--bg` 背景，`--border` 边框，`--r` 圆角 | 搜索图标 + 输入框 + 快捷键提示 |
| 快捷操作 | 清空按钮 | 32×32px，带边框的图标按钮 |
| 发送按钮 | 高度 40px，水平内边距 20px | `--accent` 背景，白色文字 |

### 输入区详情

- 搜索图标 (16px)：`--muted` 色，位于左侧
- 输入框：13px 正文字体，`--fg` 色，placeholder "输入问题，同时发送到所有 AI..."
- 快捷键提示 "Enter ↵"：10px 等宽字体，`--muted` 色，不换行，不可选中

**Focus 态：**
- 边框变为 `--accent` 色
- 外发光：`0 0 0 3px var(--accent-dim)`

### 清空按钮

- 尺寸：32 × 32px
- 1px `--border` 边框，`--r` 圆角
- 默认：白底，`--muted` 图标色
- Hover：边框变为 `--accent`，图标变为 `--accent` 色
- 功能：清空所有面板的对话内容

### 发送按钮

- 背景：`--accent`（绿色）
- 文字：白色 12px 粗体 + 发送图标 (14px)
- 圆角：`--r` (8px)
- Hover：亮度提升 1.08，阴影 `0 2px 8px oklch(58% 0.16 145 / 0.3)`
- Active：`scale(0.97)` 按压缩放
- 发送中：`opacity: 0.7`，禁用交互

## 9. 网站编辑器

点击管理面板中的 `编辑` 或 `添加网站` 按钮后，打开网站编辑器覆盖层。左侧使用**真实 webview** 加载目标网站，用户可在其中完成登录；右侧显示配置信息和自动分析结果。

```
┌─────────────────────────────────────────────────────────────────────────────────────────────────┐
│  [浏览器预览]                                     │  编辑网站                                [×]   │
│  URL: chat.openai.com                             │  名称                                     │
│  ● 状态: 就绪                    🍪 Cookie 已保存  │  网址                                     │
│                                                   │  标识色                                   │
│  ┌─────────────────────────────────────────────┐  │                                           │
│  │                                             │  │  ┌─ 自动分析 ────────────────────────┐    │
│  │           真实网页（webview）                │  │  │  🍪 Cookie: 已保存              ✓  │    │
│  │       用户可在此登录、浏览、交互              │  │  │  📝 输入框: #prompt-textarea     ✓  │    │
│  │                                             │  │  │  📨 发送按钮: button[data-...]   ✓  │    │
│  │                                             │  │  │         [重新分析] [手动配置]       │    │
│  └─────────────────────────────────────────────┘  │  └─────────────────────────────────────┘    │
│                                                   │                                           │
│                                                   │  [保存] [取消]                              │
└─────────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 9.1 编辑器整体布局

- 编辑器为覆盖层，覆盖整个应用窗口。
- 左侧面板为**真实 webview 预览区**，加载目标网站的完整页面，用户可直接交互（登录、导航等）。
- 右侧面板为配置区，用于编辑网站名称、URL、标识色，并显示自动分析结果（含 cookie 状态）。
- 右侧顶部包含标题和关闭按钮，最底部包含保存与取消操作。

### 9.2 左侧网页预览区（webview）

- 使用 Electron `<webview>` 标签加载目标网站（非 iframe），与主面板的 webview 使用相同的 partition 隔离机制。
- **用户输入 URL 并确认后，webview 立即开始加载该网站。**
- 用户可以在 webview 中正常交互：点击链接、输入文字、完成登录流程等。
- 顶部 URL 栏展示当前网站地址和加载状态指示。
- webview 的 cookie/session 与主面板中同一网站的 webview 共享（同一 partition），确保登录态一致。

#### 预览区状态

| 状态 | URL 栏状态文字 | 内容区显示 |
|------|---------------|-----------|
| 空闲（未输入 URL） | — | 空白提示：”请输入网址并回车，加载目标网站” |
| 加载中 | `加载中...` | 居中加载动画 |
| 就绪 | `就绪` | 网站完整页面，可交互 |
| 加载失败 | `加载失败` | 错误提示 + 重试按钮 |

### 9.3 右侧配置区

| 区块 | 说明 |
|------|------|
| 标题栏 | 编辑器标题 + 关闭按钮 |
| 基本信息 | 名称、网址、标识色输入字段 |
| 自动分析 | Cookie 保存状态 + 输入框/发送按钮选择器检测 |
| 操作区 | 保存、取消按钮 |

#### 配置字段

- `名称`：网站在主面板中的显示名称。
- `网址`：目标网站地址。输入后按回车或点击确认，左侧 webview 开始加载。
- `标识色`：用于面板顶部品牌色条和消息头像的颜色。

#### 自动分析区

自动分析区位于基本信息下方、操作区上方。用户在左侧完成登录后，点击 [分析] 按钮，程序同时执行 cookie 保存和选择器检测。

**区域结构：**

```text
┌──────────────────────────────────────────────────────┐
│  自动分析                                        [分析] │
├──────────────────────────────────────────────────────┤
│                                                      │
│  🍪 登录 Cookie                                      │
│  ● 已保存（上次保存: 2026-06-11）         ✓          │
│                                                      │
│  📝 输入框选择器                                      │
│  #prompt-textarea                            ✓       │
│                                                      │
│  📨 发送按钮选择器                                    │
│  button[data-testid=”send-button”]           ✓       │
│                                                      │
│              [重新分析]   [手动配置]                    │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**分析状态：**

| 状态 | 显示内容 | 说明 |
|------|----------|------|
| 空闲 | 仅显示 [分析] 按钮 | 进入编辑器后的默认状态（未登录时按钮禁用，提示”请先登录”） |
| 分析中 | 加载动画 + “正在保存 Cookie 并分析页面结构...” | [分析] 按钮禁用 |
| 分析成功 | Cookie 状态（已保存）+ 输入框/发送按钮选择器 + [重新分析] 按钮 | 用户可确认或手动配置 |
| 分析失败 | 错误提示 + [重试] [手动配置] 按钮 | 提供降级方案 |

**结果展示：**

- **Cookie 状态**：显示”已保存”或”未保存”，已保存时附带上次保存时间。绿色 ✓ 表示成功，灰色 — 表示未保存。
- 输入框选择器：等宽字体，`--muted` 色，单行省略。绿色 ✓ 表示已检测到。
- 发送按钮选择器：同上；未检测到时显示”未检测到”并提示将回退 Enter 键发送。
- 无需展示置信度、输入框类型等技术细节。

**操作按钮：**

| 按钮 | 样式 | 行为 |
|------|------|------|
| `分析` | 主按钮（`--accent` 背景） | 保存当前 cookie + 扫描 DOM 检测选择器（需已登录） |
| `重新分析` | 次按钮（边框样式） | 重新执行 cookie 保存 + 选择器检测 |
| `手动配置` | 次按钮（边框样式） | 展开手动选择器编辑区 |
| `保存` | 主按钮（`--accent` 背景） | 保存所有配置（含 cookie、选择器、基本信息）到 SiteConfig |
| `取消` | 次按钮（边框样式） | 放弃修改并关闭编辑器 |

**手动配置展开区：**

点击 [手动配置] 后，在自动分析区下方展开编辑区域：

```text
┌──────────────────────────────────────────────────────┐
│  手动配置选择器                                         │
├──────────────────────────────────────────────────────┤
│                                                      │
│  输入框选择器（每行一个，按优先级排列）                     │
│  ┌──────────────────────────────────────────────────┐ │
│  │ #prompt-textarea                                 │ │
│  │ textarea[data-id=”root”]                         │ │
│  └──────────────────────────────────────────────────┘ │
│                                                      │
│  发送按钮选择器（可选）                                  │
│  ┌──────────────────────────────────────────────────┐ │
│  │ button[data-testid=”send-button”]                │ │
│  └──────────────────────────────────────────────────┘ │
│                                                      │
│                    [验证选择器]                        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- 输入框为 textarea，每行一个选择器，支持多个按优先级排列
- [验证选择器] 按钮对输入的选择器执行注入测试

**视效规范：**

- 分析区整体有 `--bg` 背景，`--border` 边框，`--r` 圆角，与基本信息区有 12px 间距
- 分析中状态的加载动画使用 1.2s 脉冲效果，与面板加载中动画风格一致
- Cookie 状态行使用与选择器行一致的卡片样式，左侧带绿色竖条表示已保存

### 9.4 交互行为

- **输入 URL 后回车**：左侧 webview 立即加载该网站，右侧 URL 栏同步更新。
- **在 webview 中登录**：用户正常操作，登录态保存在 webview 的 session 中。
- **点击 [分析]**：程序从 webview session 提取 cookie 保存到配置，同时扫描 DOM 检测选择器。
- `保存`：保存所有配置（cookie + 选择器 + 基本信息），更新列表和主界面面板。新面板加载时自动注入已保存的 cookie。
- `取消`：放弃修改并关闭编辑器。未保存的 cookie 不会被持久化。
- 关闭按钮：关闭编辑器返回网站管理面板。

### 9.5 可视反馈

- URL 栏状态显示：`就绪`、`加载中`、`加载失败`，以及 Cookie 状态图标。
- 加载失败时，内容区显示错误提示 + 重试按钮。
- Cookie 已保存时，URL 栏右侧显示 🍪 图标 + “Cookie 已保存”标签。
- 操作按钮在 hover 时有轻微高亮与填充反馈，保持与整体 UI 风格一致。

| 标记 | 含义 | 位置 |

3. **发送按钮禁用**：发送中 `opacity: 0.7`，`pointer-events: none`
4. **导航栏状态**：显示实时注入计数（如"注入中 2/4"），全部完成后 3 秒恢复为"N 个 AI 已连接"
5. **逐个注入**：按配置间隔（默认 200ms）依次注入，避免同时操作导致卡顿

## 13. 动画与过渡

| 动画 | 时长 | 曲线 | 触发场景 |
|------|------|------|---------|
| 消息入场 | 0.3s | `cubic-bezier(0.2, 0, 0, 1)` | 消息气泡出现时，从下方 6px 淡入 |
| 输入点脉冲 | 1.2s 循环 | 线性 | AI 正在输入时，三点依次缩放 + 透明度变化 |
| 发送闪烁 | 0.8s | ease-out | 面板收到注入时，内边框 `--accent` 色闪现后消失 |
| 布局切换 | 0.3s | `cubic-bezier(0.4, 0, 0.2, 1)` | grid-template-columns 过渡 |
| 面板 Hover | 0.2s | 默认 | 阴影和边框色变化 |
| 按钮交互 | 0.12~0.15s | 默认 | 颜色/背景切换 |

### 输入点动画细节

三个圆点 (6px) 水平间距 4px，包裹在圆角气泡内（`oklch(96% 0.003 250)` 背景 + `--border` 边框，圆角 10px）。

动画关键帧：
```
0%, 60%, 100% → opacity: 0.3; scale: 0.8
30%           → opacity: 1;   scale: 1
```

第二个点延迟 0.15s，第三个点延迟 0.3s，形成波浪效果。

## 14. 对话消息样式（原型参考）

> 注：在实际 Electron 应用中，面板内容区为 webview，显示 AI 网站原生界面。以下消息样式仅用于 HTML 原型演示，作为 Paneo 自身状态提示（如注入反馈）的参考。

### AI 消息

- 头像：26×26px，圆角 6px，`--panel-accent` 14% 透明度背景 + 同色文字
- 气泡：`oklch(96% 0.003 250)` 背景，1px `--border` 边框，圆角 10px
- 文字：12.5px，行高 1.55
- 元信息：10px 等宽字体，`--muted` 色（如 "ChatGPT · 就绪"）

### 用户消息

- 头像：26×26px，蓝灰色调
- 气泡：`--accent` 背景，白色文字，无边框
- 右对齐，元信息右对齐（如 "已发送"）
